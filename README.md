# H&M Style - Fashion Homepage Mockup

A professional and visually elegant homepage mockup for a fashion brand inspired by H&M. Built using clean HTML, CSS, and modern JavaScript without frameworks.

## Features

- **Fixed Navigation Bar** - Clean navbar with brand logo and navigation links
- **Hero Section** - Full-width hero with background video and call-to-action
- **Dynamic Product Grid** - Fetches and displays products from Supabase database
- **Gender Filtering** - Toggle between men's and women's products with highlighted active tabs
- **Real-time Loading States** - Shows loading indicators and error messages
- **Google Sign-In Integration** - Secure authentication using Google Identity Services
- **Responsive Design** - Mobile-friendly layout that works on all devices
- **Neutral Color Palette** - Professional styling with light gray, white, black, and beige tones
- **Supabase Backend** - Scalable database for product management

## Setup Instructions

### 1. Configure Supabase Backend

**Step 1: Create a Supabase Project**
1. Go to [supabase.com](https://supabase.com) and create an account
2. Create a new project
3. Wait for the project to initialize (usually takes 1-2 minutes)

**Step 2: Set Up Products Table**
In your Supabase dashboard, go to the SQL Editor and run this SQL:

```sql
-- Create products table
CREATE TABLE products (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  title VARCHAR(255) NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  image_url TEXT NOT NULL,
  gender VARCHAR(10) NOT NULL CHECK (gender IN ('men', 'women')),
  description TEXT,
  category VARCHAR(100),
  sizes TEXT[],
  colors TEXT[]
);

-- Enable Row Level Security (RLS)
ALTER TABLE products ENABLE ROW LEVEL SECURITY;

-- Create policy to allow public read access
CREATE POLICY "Allow public read access on products" ON products
  FOR SELECT USING (true);
```

**Step 3: Configure Supabase Storage (Optional)**
If you want to upload your own product images:
1. Go to Storage in your Supabase dashboard
2. Create a new bucket called `product-images`
3. Make it public by updating the bucket policy
4. Upload your images and use the public URLs in your products table

**Step 4: Configure Your Credentials Securely**
1. Go to Settings > API in your Supabase dashboard
2. Copy your Project URL and anon public key
3. **IMPORTANT**: Never commit credentials to git! Choose one of these secure methods:

**Method A: Local Config File (Recommended for Development)**
```bash
# Copy the example config file
cp js/config.example.js js/config.js

# Edit js/config.js with your actual credentials
# This file is already in .gitignore so it won't be committed
```

**Method B: Environment Variables (Production)**
For deployment, use environment variables and a build process to inject them.

**Method C: Server-Side Proxy (Most Secure)**
Create a backend API that handles Supabase calls to keep credentials server-side.

### 2. Add Brand Assets
Replace the placeholder files in `/assets/`:
- `logo.png` - Your brand logo
- `hero.mp4` - Hero section background video (or `hero.jpg` for fallback)
- Create `placeholder-product.jpg` for products with missing images

### 3. Configure Google Sign-In
To enable Google authentication:

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select an existing one
3. Enable the Google+ API
4. Create OAuth 2.0 credentials (Web application)
5. Add your domain to authorized origins
6. Copy your Client ID
7. Replace `YOUR_GOOGLE_CLIENT_ID` in both `login.html` and `auth.js` with your actual Client ID

### 4. Launch the Website

**For Local Development:**
```bash
# Method 1: Using npm (recommended)
npm run dev

# Method 2: Manual setup
cp js/config.example.js js/config.js
# Edit js/config.js with your credentials, then:
python -m http.server 8000

# Method 3: Using Live Server (VS Code extension)
# Right-click index.html → "Open with Live Server"
```

**For Production Deployment on Vercel:**
1. **Fork/Clone to GitHub** - Push your code to a GitHub repository
2. **Connect to Vercel** - Go to [vercel.com](https://vercel.com) and import your GitHub repo
3. **Set Environment Variables** in Vercel dashboard:
   - `SUPABASE_URL` = `https://your-project-id.supabase.co`
   - `SUPABASE_KEY` = `your-anon-public-key`
   - `GOOGLE_CLIENT_ID` = `your-google-client-id` (optional)
4. **Deploy** - Vercel will automatically build and deploy your site

**Vercel Deployment Steps:**
```bash
# 1. Install Vercel CLI (optional)
npm i -g vercel

# 2. Deploy from command line (or use GitHub integration)
vercel

# 3. Set environment variables
vercel env add SUPABASE_URL
vercel env add SUPABASE_KEY
vercel env add GOOGLE_CLIENT_ID

# 4. Redeploy with environment variables
vercel --prod
```

**Environment Variables in Vercel Dashboard:**
1. Go to your project settings in Vercel
2. Navigate to "Environment Variables"
3. Add these variables:
   - **Name:** `SUPABASE_URL`, **Value:** `https://your-project.supabase.co`
   - **Name:** `SUPABASE_KEY`, **Value:** `your-anon-public-key`
   - **Name:** `GOOGLE_CLIENT_ID`, **Value:** `your-google-client-id`
4. Redeploy the project

### 5. Test the Integration
1. Open http://localhost:8000 in your browser
2. The page should load with "Women" tab active by default
3. You should see sample products from your Supabase database
4. Click "Men" tab to switch to men's products
5. Check browser console for any errors

## Code Architecture

### JavaScript Principles
- **Modular & Functional** - No classes, small reusable functions
- **Declarative Code** - Clear, readable function names
- **Descriptive Variables** - Uses auxiliary verbs (isUserLoggedIn, hasError, etc.)
- **Separation of Concerns** - Auth, utilities, and main logic in separate files

### CSS Features
- **CSS Custom Properties** - Consistent color palette and spacing
- **Mobile-First Design** - Responsive breakpoints at 768px and 480px
- **Smooth Animations** - Subtle transitions and hover effects
- **Grid Layout** - Modern CSS Grid for product display

### HTML Structure
- **Semantic HTML5** - Proper use of header, main, section, footer
- **Accessibility** - Alt text, proper heading hierarchy
- **Clean Markup** - Minimal, semantic structure
